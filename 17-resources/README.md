# 17. Resources

A good resource list is not random. It is curated to reduce search time during the build.

## Resource categories

- documentation
- templates
- APIs
- UI kits
- deployment docs
- pitch guides
- open data
- communities

## What to keep in your personal stack

- a notes file,
- a prompt library,
- a deployment checklist,
- an idea scorecard,
- and a backup demo folder.

## Resource habit

Do not keep re-searching the same things during every hackathon.  
Turn useful links into a system.
